编译Android内核时找不到openssl/bio.h

2019-06-24 编译 阅读量

参考：https://blog.csdn.net/qq_41533289/article/details/82985788

环境：Ubuntu 16.04 + Android P

最近重装了系统，从14.04换成了16.04，之后按照官网安装了Android的编译环境，但是在编译Android P的项目时内核部分出现了下面的错误：

fatal error: openssl/bio.h: No such file or directory

代码里找不到什么问题，上网搜索得知出现fatal error: openssl/×××.h: No such file or directory。是因为没有安装libssl-dev，libssl-dev包含libraries, header files and manpages，他是openssl的一部分，而openssl对ssl进行了实现～

解决方案：

使用sudo apt-get install libssl-dev来安装libssl-dev即可
